API测试思维

概述：

接口测试：又称为API测试，是软件测试的一种测试模式，它包含了两个维度。
分别为: 狭义、广义
在狭义的⻆度上指的是: 对应用程序接口的功能进行测试，
在广义的维度上是指: 集成测试中通过调用API测试整体的功能的完成度，可靠性，安全性和性能。

金字塔模型:

金字塔的模型中: 测试分为三个维度来进行思考，分别是单元，服务、UI三个层级。

Service Test测试用例的编写：

例1：接口无请求参数
接口信息：
Doc: 获取用户初始化信息
URL：smart/getInitInfo
Params: None
接口分析：
接口业务： 设备信息、用户信息、固件升级弹框、心率设置
接口请求参数： 无请求参数
接口返回值应用场景:      我的-我的设备
                      我的-个人信息
                      首页-心率设置
                      首页-固件升级
getInitInfo返回值

{"rtn": 0, "data": {

"userExtInfo": {}, "hr": {}, "devices": [], "global": {}, "firmwareInfo": {} } }

getInitInfo用例设计：
    1.使用updateUserExtInfo接口提交用户信息，校验userExtInfo字段返回值正确性
    2.使用updateUserExtInfo接口提交用户心率设置，校验hr字段返回值正确性
    3.使用bindDevice接口提交信息，校验devices字段返回值正确性
    4.使用cms提交固件信息，校验firmwareInfo字段返回值正确性
例2：接口有请求参数
接口信息：
Doc：绑定设备
URL：bindDevice
Params：sn         string     必填
        bindType   int        非必填
        name       string     非必填
接口分析
接口业务：设备信息、用户配置
接口参数： sn、bingType、name
接口场景：
            我的-我的设备
            首页-设备信息
            我的-设备信息
接口返回值

{ "rtn": 0, "data": { "boundDeviceSN": "80170756", "config": { "hrMeasurement": 1, "hrLimitWarning": 0 } } }

bindDevice用例设计
1.提交必填参数sn，使用getInitInfo接口返回值校验sn是否提交成功
2.提交组合参数sn+name+ bindType ，使用getInitInfo接口返回值校验sn+name+ bindType是否提交成功
3.提交参数sn，使用updateHRDeviceSettings更新用户设置 校验：bindDevice返回值
